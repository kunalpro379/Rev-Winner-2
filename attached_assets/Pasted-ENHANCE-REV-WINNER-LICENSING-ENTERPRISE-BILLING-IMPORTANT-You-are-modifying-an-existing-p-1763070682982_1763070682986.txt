2. SUBSCRIPTION & PACKAGES (INDIVIDUAL USERS)

Platform Access gives the user the right to use the platform at all.
Implement these products/plans:

1. Month-to-Month (M2M)

Price: $40 per user / 30 days

After 30 days, if not renewed:

User can still sign in

Immediately show blocking popup:

> “Your subscription has expired. Please renew to continue using Rev Winner.”



Prevent access to core app until they renew




2. Long-Term Access (One-Time Non-Renewing Plans)

6 Months → $200

12 Months → $399

36 Months → $499

Store start_date and end_date for each subscription

When end_date < now → treat as expired just like M2M




> Implementation:

Create/extend a subscriptions table to include: user_id, plan_type (m2m, 6m, 12m, 36m), start_date, end_date, status, and razorpay_subscription_id or razorpay_order_id.

At login and on each main app load, check status and end_date.

⏱ 3. MANDATORY ADD-ON: SESSION MINUTES

All users (individual or enterprise) need session minutes to use AI features in the web assistance (TTS/AI processing).

Price: $8 per 500 minutes

Tiers: 500, 1000, 1500, 2000, 3000, 5000 minutes

Validity: 31 days from purchase

No rollover across periods


Logic:

Track minutes_purchased, minutes_used, purchase_date, expiry_date per user (and/or per enterprise where applicable)

When minutes_used >= minutes_purchased OR expiry_date < now:

Block AI features

Show message:

> “Your Session Minutes are exhausted. Please purchase an add-on to continue.”




Allow buying any tier at any time; new purchase can either:

Extend validity from current date, and

Add minutes to remaining pool or reset (choose the simpler, well-structured model and implement consistently; document in code)



> Create/extend session_minutes table with id, user_id (or enterprise_id if pooled), minutes_purchased, minutes_used, purchase_date, expiry_date.

4. OPTIONAL ADD-ON: TRAIN ME

Train Me lets users / enterprises upload internal content for personalized responses.

Pricing

$20 per user per month (or enterprise-wide with per-user toggles)


Restrictions

Max 100 documents per user (or per enterprise, but with per-user domain mapping)

Supported formats: pdf, docx, xlsx, csv, txt, URLs, jpg, png

Up to 5 domain categories (e.g., “Product”, “Pricing”, “Processes”, etc.)


Behavior

AI should:

1. First query Train Me docs


2. If inadequate, fall back to global web/LLM knowledge


3. Prioritize Train Me’s knowledge in responses




UI/Access Logic

If Train Me Add-On not purchased:

Train Me toggles in License Manager / user settings should be disabled

Tooltip:

> “Train Me Add-On not purchased. Please ask your License Manager to upgrade.”




If Train Me purchased:

License Manager can enable/disable Train Me per user

If disabled for a user → that user shouldn’t see Train Me tab / upload options



> DB:

train_me_addon table: id, user_id or enterprise_id, active, renewal_date, file_count, domain_categories (JSON), metadata.

Separate table for documents if not already: train_me_documents.

🤖 5. OPTIONAL ADD-ON: REV WINNER DAI (INTERNAL OPENAI ENGINE)

Rev Winner DAI is for users who do NOT want to bring their own API key.

Packages (with ~30% margin already baked in)

Monthly recurring plans:

Lite → 1M tokens → $10/month

Moderate → 3M tokens → $24/month

Professional → 6M tokens → $45/month

Power → 12M tokens → $78/month

Enterprise → 25M tokens → $156/month


Refill one-time packs:

1M → $10

3M → $24

6M → $45


Behavior

1. On DAI purchase:

Generate a unique internal OpenAI API key or token mapping for that user or enterprise

Store securely (encrypted) in DB

Mark user/enterprise as using DAI instead of “Bring Your Own Key”



2. When tokens are exhausted OR 30 days pass:

Individual Users:

Show message:

> “Your Rev Winner DAI tokens have expired. Renew your DAI Add-On or add your own API key under Settings.”



In settings:

DAI option stays visible but disabled until renewed

Provide field to paste own API key as alternative



Enterprise Users:

Show message:

> “Your organization’s DAI tokens are exhausted. Please contact your License Manager to renew DAI.”






3. If DAI is not purchased and user chooses “Rev Winner DAI” in settings:

Show popup/toast:

> “Rev Winner DAI is not part of your current plan. Please purchase the DAI Add-On or contact your License Manager.”






Settings UI – AI Engine Section

Option 1: “Use My Own API Key”

Input field for key


Option 2: “Use Rev Winner DAI”

Enabled only if DAI Add-On is active



> DB:

dai_packages: id, user_id or enterprise_id, tokens_total, tokens_used, package_name, billing_type (monthly/refill), start_date, expiry_date.

ai_usage_logs table for logging usage per user and per feature.

6.2.1 Assign License to User

Flow:

1. License Manager clicks “Assign License”.


2. Inputs:

User name

User email



3. System creates enterprise_user_map row with status pending_activation.


4. System sends email to the user:

Subject: “Your Rev Winner License Has Been Assigned”

Body includes:

“You’ve been granted access to Rev Winner under [Org Name]. Click below to activate your account.”

Button: [Activate My Account] with token link




5. Activation link opens a page:

Validate token & expiration

Force user to set a new password (even if email already exists with no password)

After password setup → mark status active

Grant access to platform, subject to subscription & add-ons




6.2.2 Swap Licenses Between Users

License Manager can select User A → “Revoke License” and assign that freed seat to User B.

User A → inactive (cannot log in).

User B → new activation email & flow.


6.2.3 Enable / Disable Access

Toggle switch:

Disable → user cannot log in, show “Access disabled by your organization” if they try.

Enable → restore login.



6.2.4 Train Me Control

If Train Me Add-On is purchased at enterprise level:

Per-user toggle: “Train Me Access”

ON → show Train Me section in user UI

OFF → hide it & disallow document usage for that user


If not purchased:

All toggles greyed out with tooltip:
“Train Me is not enabled for your organization.”



6.2.5 DAI Control

License Manager can see:

DAI status: Active / Inactive

Tokens remaining

Renewal date

Button: “Renew DAI” (takes them to billing/payment)


For DAI-active enterprises:

All users automatically use enterprise DAI unless they explicitly set “My Own API Key” in settings.


For DAI-inactive enterprises:

“Rev Winner DAI” option appears but disabled with message: “Your organization has not enabled DAI. Talk to your License Manager.”
