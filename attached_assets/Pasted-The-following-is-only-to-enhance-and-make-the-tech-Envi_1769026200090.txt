The following is only to enhance and make the tech Environment functional, no other changes needed. Also change the name of the section from Tech Environment to Map/Flow

Objective:
Generate a visual Tech Environment Mind Map / Workflow that helps a seller clearly understand the customer’s IT stack, infrastructure, workflows, dependencies, gaps, and pain points, even when the customer provides partial, vague, or non-technical information during a sales conversation.
Context Awareness Rules:
•	Analyze the live conversation or provided transcript
•	Always remain aware of the user’s domain expertise in the rev winner web app.
•	Prioritize train me addon information for generating a response, if no relevant information found generate information from universal knowledge base of the AI currently in use.
•	If repeat scenario, use previous responses instead of utilizing excessive tokens through AI.
•	Fast response needed, 
•	Infer missing components using industry intelligence and best practices
•	Stay strictly contextual to what the customer is discussing
•	Never overwhelm—prioritize clarity over completeness
________________________________________
🔹 Input Signals
•	Live conversation transcript (if available)
•	Seller-selected Domain Expertise (e.g., IT Infra, Cybersecurity, Cloud, IT Development, MSP, SaaS, Microsoft, Cisco, NetApp, Dell etc.)
•	Optional seller intent:
o	“Understand current environment”
o	“Identify pain points”
o	“Prepare solution recommendation”
o	“Compare current vs future state”

o	Applications
o	Data

•	Continuously refine assumptions as conversation evolves
